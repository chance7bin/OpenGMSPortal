    Model Classification

    According to the national standard，We designed a set of geographic analysis model metadata to describe the characteristics of a geographic analysis model, such as classification identification information, scope of application, model performance, parameter types, operating conditions, modeling principles, solution methods, integration mode and so on. The metadata applies to the areas of geographic analysis models’ computation, integration, sharing and services.
    According to the earth system science classification system, we classify the geologic analysis models, and the geologic analysis model classifies the models according to their contents, modeling methods, and organizational levels. The following is a classification system based on the earth system science classification system. There are three levels of directories and you can search and browse the model according to the type of model you want to know.

    Computable model can support user to upload their own data, invoke model service and get result. The content below show you how to invoke model service step by step.

    Choose computable model

    To invoke model service, first check the computable model list. On the menu the link is on model -> model instance -> computable model. The list page have all the computable model and its simple description and you can choose one model to see the further information.

    Select computable server

    In the detail page, you can get computable model information, environment information, availible server information.To invoke a model service, please ensure you have already logged in first then select one of the availible server.

    Config and execute model

    In the execution page, there are two function panel. one is diagram panel, model is divided into different ellipses which we called states, you can click on the state to view the state information panel on the right. The panel describe the model behaviour and the structure of model input and output data.

    By click the invoke button on the left, right function panel will change to invoke panel. In this panel input data should be uploaded and output data should be given a filename. To have a quick try of running model, you can select test data by clicking the test button. After all of the data prepared, click the start button to run the model. You can stay on the page to waiting for the result or you can leave this page and check the result in your userspace.

    Get model output data

    Click on the upper right corner to jump to userspace page. All of your model invoke records are in the task panel. You can search the model name or check the create time of task to locate your task, and download the output files.

    Definition

    Service container is a lightweight software which publish geo-analysis models to services in the web environment and manage them. It can be downloaded in the portal site of the OpenGMS. In the package, there is a console that can start the container.


    Components Introduction

    The service container of geo-analysis models consist of three components, including service container console, service container and database. Service container is the most important component of the system. Database is make up by MongoDB and it is used to store the information of model services, records and data. Service container console is a tool to start database and service container.

    Desktop

    Using OGMS desktop software, you can invoke model services from the container which is in your local area network or in internet. In order to deal with the big data of model, you can deploy model container on the computer in your local area network and data don't need to transmit by internet. also you can manage running model and visual the model result by destop.

    Search Server

    Click server button on the left menu to go to the server panel. This panel shows the all of the container which have connected to desktop, you can add your own container or delete other container.

    Get Services

    Click ModelRes or DataRes button on the left menu to go to the model or data service panel. This panel show all of the service from the container desktop connected. you can check the service detail information or add to your favor list by the button on the service item.

    Invoke Services

    Click favor button on the left menu to go to the favor panel. Select the service you want to invoke and go to config the service.

    In the config service, you check the diagram of model state information and prepare your data just as in the portal. You can save your configur

    Manage Task

    Click task button on the left menu to go to the task panel. There are three state of task. To run task is the task which is ready to run. Running task is the rate of task progress. Finish task are tasks which have finished and are able to download its result data or view result.
